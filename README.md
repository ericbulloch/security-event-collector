# Security Event Collector

A lightweight log-tailing agent that reads security events from OS log files,
parses them into the format expected by the
[security-event-correlator](https://github.com/ericbulloch/security-event-correlator),
and forwards them to its ingestion API.

When the correlator API is unreachable, events are buffered locally in a SQLite
database and flushed automatically once the API comes back online.

---

## How it works

```
Log files / Windows Event Log
        ↓  (poll every 2 s)
   File position tracking (SQLite)
        ↓
   Per-platform parsers → SecurityEvent dicts
        ↓
   Local SQLite buffer  ←── used when API is down
        ↓
   HTTP POST  →  correlator /v1/events
```

Key properties:
- **No data loss** — events buffer in SQLite if the API is unreachable
- **Restart-safe** — byte offsets are persisted; replaying old log lines on restart is avoided
- **Rotation-aware** — detects `logrotate` (inode change on Linux) and file truncation
- **Cross-platform** — Linux and Windows in one codebase, no platform-specific dependencies

---

## Requirements

- Python 3.8+
- A running instance of [security-event-correlator](https://github.com/ericbulloch/security-event-correlator)
- An API key from the correlator (generated by `python scripts/seed_dev.py`)

## Getting started

### 1 — Clone and install dependencies

```bash
git clone https://github.com/ericbulloch/security-event-collector
cd security-event-collector
pip install -r requirements.txt
```

### 2 — Configure

```bash
# Linux / macOS
cp .env.example .env

# Windows
copy .env.example .env
```

Open `.env` and set at minimum:

```
COLLECTOR_API_URL=http://<correlator-host>:8000
COLLECTOR_API_KEY=<key from seed_dev.py>
```

The `COLLECTOR_SOURCE` defaults to `socket.gethostname()`.  Override it if
the hostname is not meaningful (containers, cloud VMs with generated names).

### 3 — Run

```bash
python main.py
```

The collector will log which files it is watching and report each flush:

```
2026-07-08T10:00:00  INFO      src.collector  Collector started  source=web-01  api=http://localhost:8000  poll=2s
2026-07-08T10:00:00  INFO      src.collector  Watching: /var/log/auth.log
2026-07-08T10:00:00  INFO      src.collector  Watching: /var/log/ufw.log
2026-07-08T10:00:02  INFO      src.sender     Flushed 3 event(s) to correlator
```

## Platform details

### Linux

| Log file | Events detected |
|---|---|
| `/var/log/auth.log` | SSH failed login, SSH successful login, sudo execution |
| `/var/log/ufw.log` | UFW BLOCK / ALLOW (network connections) |
| `/var/log/audit/audit.log` | Sensitive file access (open/openat syscalls to `/etc/shadow`, `id_rsa`, etc.) |

The collector must have **read access** to these files.  On most systems:
```bash
sudo usermod -aG adm $USER          # gives access to auth.log and ufw.log
sudo usermod -aG audit $USER        # gives access to audit.log (may vary by distro)
```

Or run as root for testing:
```bash
sudo python main.py
```

### Windows

The collector uses `wevtutil` (built into Windows — no extra tools needed) to
query the **Security Event Log** for these event IDs:

| Event ID | Description | Maps to |
|---|---|---|
| 4625 | Failed logon | `login_attempt / failed` |
| 4624 | Successful logon (network types only) | `login_attempt / success` |
| 4672 | Special privileges assigned | `privilege_change / elevated` |
| 4663 | Object access | `file_access / accessed` |
| 5156 | Network connection permitted | `network_connection / allowed` |

The collector also tails the **Windows Firewall log** if it exists.

#### Enable audit policies (run in an elevated PowerShell or cmd)

Some event IDs require audit policy to be turned on:

```powershell
# Enable object access auditing (needed for 4663 — file access)
auditpol /set /subcategory:"File System" /success:enable /failure:enable

# Enable process tracking (network connections — needed for 5156)
auditpol /set /subcategory:"Filtering Platform Connection" /success:enable

# Enable logon auditing (4624, 4625 are usually on by default)
auditpol /set /subcategory:"Logon" /success:enable /failure:enable
```

#### Enable Windows Firewall logging

```powershell
netsh advfirewall set allprofiles logging droppedconnections enable
netsh advfirewall set allprofiles logging allowedconnections enable
```

#### Running on Windows

```powershell
# Must be run as Administrator to read the Security event log
python main.py
```

## Configuration reference

All settings use environment variables (loaded from `.env`).

| Variable | Default | Description |
|---|---|---|
| `COLLECTOR_API_URL` | `http://localhost:8000` | Correlator API base URL |
| `COLLECTOR_API_KEY` | _(required)_ | API key from correlator seed script |
| `COLLECTOR_SOURCE` | hostname | Value for the `source` field on all events |
| `COLLECTOR_POLL_INTERVAL` | `2` | Seconds between poll cycles |
| `COLLECTOR_BATCH_SIZE` | `25` | Events per flush pass (stay under rate limit) |
| `COLLECTOR_REQUEST_TIMEOUT` | `10` | Per-request HTTP timeout (seconds) |
| `COLLECTOR_DB_PATH` | `collector.db` | Path to local SQLite buffer |
| `COLLECTOR_LOG_LEVEL` | `INFO` | `DEBUG`, `INFO`, `WARNING`, or `ERROR` |
| `COLLECTOR_AUTH_LOG` | `/var/log/auth.log` | Linux: SSH / sudo log |
| `COLLECTOR_UFW_LOG` | `/var/log/ufw.log` | Linux: UFW firewall log |
| `COLLECTOR_AUDIT_LOG` | `/var/log/audit/audit.log` | Linux: auditd log |
| `COLLECTOR_WINDOWS_FIREWALL_LOG` | `%SystemRoot%\...\pfirewall.log` | Windows Firewall log |

## Running as a service

### Linux (systemd)

Create `/etc/systemd/system/sec-collector.service`:

```ini
[Unit]
Description=Security Event Collector
After=network.target

[Service]
Type=simple
User=collector
WorkingDirectory=/opt/security-event-collector
EnvironmentFile=/opt/security-event-collector/.env
ExecStart=/usr/bin/python3 /opt/security-event-collector/main.py
Restart=on-failure
RestartSec=5

[Install]
WantedBy=multi-user.target
```

```bash
sudo systemctl daemon-reload
sudo systemctl enable --now sec-collector
sudo journalctl -fu sec-collector
```

### Windows (Task Scheduler)

```powershell
$action  = New-ScheduledTaskAction -Execute "python" -Argument "C:\collector\main.py" -WorkingDirectory "C:\collector"
$trigger = New-ScheduledTaskTrigger -AtStartup
$settings = New-ScheduledTaskSettingsSet -RestartOnIdle -ExecutionTimeLimit (New-TimeSpan -Hours 0)
Register-ScheduledTask -TaskName "SecurityEventCollector" -Action $action -Trigger $trigger -Settings $settings -RunLevel Highest
```

## Adding a new log source

1. Create `src/parsers/my_log.py` extending `BaseParser` with a `parse(line) -> Optional[dict]` method.
2. In `src/collector.py`, import your parser and append a `FileTailer` to `self.tailers` in the appropriate setup method.
3. Add the log path to `src/config.py` and `.env.example`.
